11. Known GHL Constraints

Opportunity list view: can't show custom scores. Contacts list: can. Kanban: sorts by stage only. List customizations: browser-specific, not backend-saved.
GHL branches do NOT rejoin/merge. Consequence: a per-touch If/Else pattern dead-ends every branch except the one carrying the sequence — this is what broke the first two Hot builds. Tier cadences are therefore built as ONE linear chain gated by a SINGLE If/Else at the top (see §14b). Any tail action (e.g. demotion) that must run on more than one branch has to be DUPLICATED per branch, not shared.

12. AI Layer — Analyze / Draft / Route / Vet, Never Commit

AI never makes offers, answers valuations, or sends committal messages unsupervised. Scope: canned + AI-drafted replies (human-approved), tough-question deflection to calendar/form, RVM drops, lead vetting.

14b. Tier Mail-Cadence Workflows — BUILT & PUBLISHED ✅

Three GHL workflows, each triggered by its bucket tag. Separate from Seller 0 — Seller 0 stays its own trigger-less digital engine (enrolled manually/bulk); the mail workflows are standalone and coexist. No coupling.

ARCHITECTURE (all three): ONE linear chain gated by a SINGLE If/Else at the top — NOT a per-touch check (see §11 for why). Structure:
- Trigger: Contact Tag added (the tier tag).
- If/Else "Mail Automation OFF": custom value mail_automation_enabled Is off → OFF branch carries the whole sequence. None branch → empty (END) for now; this is the toggle-ON / vendor path, wired at toggle-ON day (§14c).
- On the OFF branch: a linear chain of Create Task → Wait → Create Task → Wait ... → demotion tail.
- Each Create Task: assigned to Brad, due 0 days/today, 8:00 AM. (Cloned tasks drop due dates — always re-verify red dots and re-set 0-days.)

Live workflow names in GHL (exact):
- "Sequence - Hot Tag Mail"
- "Sequence - Warm Tag Mail"
- "Sequence - Low Tag Mail"

HOT (trigger tag hot): 8 touches, every 2 weeks (~4-month window), alternating PRIMARY / POSTCARD (T1 PRIMARY, T2 POSTCARD ... T8 POSTCARD). Tail: Remove hot → Add warm. PUBLISHED.
WARM (trigger tag warm): 4 touches, monthly (30-day waits, incl. one after T4; ~4-month window), alternating PRIMARY / POSTCARD. Tail: Remove warm → Add low. PUBLISHED.
LOW (trigger tag low): 4 touches, every 2 months (~8-month window), POSTCARD only. Tail: Remove low ONLY — no Add-tag. Exits mail; contact stays in Seller 0 unless moved to LTN. PUBLISHED.

Demotion chain: Hot (~4mo) → Warm (~4mo) → Low (~8mo) → out = ~16 months total if never engaged. Time-based, handled in the workflows — the scoring function never moves tiers.

Task-based, not slot-tags: each mail touch creates a transient TASK (the "who to mail" list for the OFF/manual path), not a permanent slot tag. Contact only ever carries its one tier tag. At toggle-ON, task steps become vendor mail actions.

Settings per workflow: re-entry should be OFF (a re-fired tier tag must not restart a cadence mid-run). Skip-weekends left OFF on tasks (manual mail-prep task landing on a weekend just waits till Monday).

Recurring GHL AI builder issues (confirmed again this build): (a) refuses to build a clean linear chain — keeps reverting to per-touch branching (build "bones only, no If/Else" first, then insert the single gate manually); (b) off-branch tasks build empty; (c) due dates don't stick (manual 0-days fix); (d) steps sequence out of order; (e) edits to an existing tangled workflow throw network errors — cleaner to rebuild from empty than to patch. Verify each after every AI build.

14d. Mail-Exit on Offer — WIRED ✅

Rule: a contact is removed from ALL mail paths the moment an offer is logged — NOT on mere response. A reply is noisy (wrong number, "not now"); an offer (pipeline stage) is the objective "real deal" signal.
Implemented inside Seller 7 - Offer Sent (which already triggers on Seller Leads Pipeline → "Seller Offer Sent" stage). Two actions added at the TOP of Seller 7, immediately after the trigger, before "Send Notification":
  1. Remove from Workflow → "Sequence - Hot Tag Mail", "Sequence - Warm Tag Mail", "Sequence - Low Tag Mail", "Seller 0 - Cold Seller Outreach" (by name — NOT "all"; must not remove from Seller 7 itself).
  2. Add Contact Tag → offer-made.
Remove-from-Workflow stops FUTURE touches but does not delete tasks already created. The offer-made tag is the guard: FILTER OUT offer-made on every manual mail export so a live-offer contact is never mailed.
Exact stage string: "Seller Offer Sent" (not "Offer Made").
